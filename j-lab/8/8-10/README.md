## 1차원(1D) 차분

배열에 [l, r] 구간에 +k를 하고 싶을 때

```py
diff[l] += k
diff[r+1] -= k
```

그 이유는 r+1부터 다시 -k를 해서 앞에서 더한 효과를 끊어주는 것
즉, 누적합을 돌리면 [l, r] 구간까지만 +k가 유지되고, r+1부터는 0으로 돌아감.

## 2차원(2D) 차분

이제 “행”과 “열” 두 방향으로 각각 누적합을 돌리게 된다.
즉, 어떤 직사각형에 +k를 하려면
-> 왼쪽 위 모서리에서 +k를 시작하고
-> 그 영향이 오른쪽, 아래쪽으로 “퍼지되”,
-> “끝나는 지점”에서는 다시 반대로 끊어줘야 함.

| 위치         | 역할                        | 부호 |
| ------------ | --------------------------- | ---- |
| (i1, j1)     | 시작점 (여기서부터 +k 시작) | +k   |
| (i1, j2+1)   | 가로 방향으로 효과 끊기     | -k   |
| (i2+1, j1)   | 세로 방향으로 효과 끊기     | -k   |
| (i2+1, j2+1) | 두 번 끊긴 부분 복구        | +k   |

### 왜 마지막이 다시 +k 인가?

(i2+1, j2+1) 지점은 가로로 한 번(-k), 세로로 한 번(-k) 두 번 빠짐.

그런데 실제론 한 번만 끊어야 하므로
“한 번 되돌려주기 위해” 다시 +k를 해줘야함

즉, 이중 보정(두 번 빼졌으니 한 번 더하기) 개념이다.
